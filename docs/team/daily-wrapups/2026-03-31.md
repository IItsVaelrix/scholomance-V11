### 1. Symmetry AMP Microprocessor System

**Created two specialized microprocessors:**

| Microprocessor | ID | Purpose | Stage |
|---------------|-----|---------|-------|
| **Symmetry AMP** | `amp.symmetry` | Detect/analyze/suggest symmetry | Post-decode |
| **Coord Symmetry AMP** | `amp.coord-symmetry` | Transform/orient/enforce coordinates | Pre-render |

**Why the split?**
- Detection (eye) ≠ Transformation (hand)
- Different inputs, outputs, failure modes, caching strategies
- Clean architecture over swollen chimera

**Symmetry Types Supported:**
- ✅ Vertical mirror (left ↔ right around detected axis)
- ✅ Horizontal mirror (top ↔ bottom)
- ✅ Radial rotation (90°, 180°, 270°)
- ✅ Diagonal mirror (swap X ↔ Y)

**Transform Modes:**
- `overlay` — Keep originals + add mirrors
- `replace` — Only transformed coords
- `canonicalize` — Rebuild full symmetric set from governing side

**Overlap Policies:**
- `prefer-original` — Keep original on collision
- `prefer-transformed` — Keep mirror on collision
- `max-emphasis` — Keep highest emphasis
- `blend` — Average colors

**Grid Snapping:**
- `coordinateSpace: 'pixel'` — Snap to pixel grid
- `coordinateSpace: 'cell'` — Snap to cell units

## Key Architectural Decisions

### 1. Detection ≠ Transformation

**Decision:** Split `amp.symmetry` (detect) from `amp.coord-symmetry` (transform)

**Rationale:**
- Different input/output contracts
- Different failure modes
- Different caching strategies
- Different UI expectations
- Prevents swollen chimera anti-pattern

**Quote:** *"One is the eye. The other is the hand."*

---

### 2. Bytecode Is Canonical

**Decision:** All persistent state, exports, and interoperable data use bytecode encoding

**Rationale:**
- AI-parsable error communication
- Deterministic error encoding
- Checksum integrity verification
- Cross-system interoperability
- Automated debugging support

**Impact:** All agents must now use bytecode assertions in QA tests.

---

### 3. Render-Time Symmetry (Not Upload-Time)

**Decision:** Apply symmetry transforms at render time, with optional bake-at-export

**Rationale:**
- Preserves flexibility (toggle on/off)
- Non-destructive workflow
- Experimentation friendly
- Export-time baking for final assets

**Pipeline:**
```
upload → detect only (metadata)
preview → transform optionally (live toggle)
export → bake if requested (canonical)
```

---

### 4. Axis-Based Transforms (Not Center-Based)

**Decision:** Use detected symmetry axis, not canvas center

**Rationale:**
- Canvas center ≠ content center
- Detected axis is visually accurate
- Prevents crooked renders
- Mathematically correct AND visually centered

**Code:**
```javascript
const axisX = symmetry.axis?.x ?? dimensions.width / 2;
const axisY = symmetry.axis?.y ?? dimensions.height / 2;
```

## Lessons Learned

### 1. Web Workers Don't Exist in Test Environment

**Problem:** `processorBridge.execute()` failed in tests with "Worker is not defined"

**Solution:** Import direct function for tests:
```javascript
import { runCoordSymmetryAmp } from '../../codex/core/pixelbrain/coord-symmetry-amp.js';
```

---

### 2. Bytecode Parsing with Dashes in Base64

**Problem:** Base64 context can contain `+` and `/` which get URL-encoded

**Solution:** Use URL-safe base64:
```javascript
const contextB64 = Buffer.from(JSON.stringify(context))
  .toString('base64')
  .replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
```

---

### 3. Overlap Resolution Needs Explicit Policy

**Problem:** Mirrored coordinates can land on same position as originals

**Solution:** Four explicit policies:
- `prefer-original` — Deterministic, preserves source
- `prefer-transformed` — Use mirror on collision
- `max-emphasis` — Keep highest emphasis value
- `blend` — Average colors

**Default:** `prefer-original` (most predictable)
