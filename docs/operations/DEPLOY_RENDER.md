# Deploying Scholomance on Render

This project is configured for single-domain app hosting:
- React app (Vite build output) is served by Fastify in production.
- API routes remain under `/api/*` and `/auth/*`.
- Sessions use Redis.
- User data uses SQLite on a persistent disk.

## 1. Prerequisites

- Render account
- Upstash account (for Redis)
- Git repo connected to Render

## 2. One-click Blueprint Deploy

Use the root `render.yaml` blueprint.

It provisions:
- `scholomance-app` (Docker web service)
- persistent disk mounted at `/var/data`

**Note:** You must provide your own `REDIS_URL` (e.g., from Upstash) in the Render dashboard after initial setup.

## 3. Upstash Redis Setup

1. Create a Redis database on [Upstash](https://console.upstash.com/).
2. Copy the **Redis Connect URL** (formatted as `redis://default:password@endpoint:port`).
3. Add this URL as `REDIS_URL` in your Render Environment Variables.

## 4. Required Environment Variables

The blueprint or manual setup requires:
- `NODE_ENV=production`
- `HOST=0.0.0.0`
- `TRUST_PROXY=true`
- `USER_DB_PATH=/var/data/scholomance_user.sqlite`
- `COLLAB_DB_PATH=/var/data/scholomance_collab.sqlite`
- `AUDIO_STORAGE_PATH=/var/data/audio`
- `RHYME_ASTROLOGY_OUTPUT_DIR=/var/data/rhyme-astrology` (required when `ENABLE_RHYME_ASTROLOGY=true`)
- `ENABLE_COLLAB_API=false` (enable only if intentionally exposing collab API)
- `AUDIO_ADMIN_TOKEN` (required; set a long random secret)
- `REDIS_URL` (from Upstash)
- `SESSION_SECRET` (auto-generated by blueprint)

## 5. Verification

You can verify your Upstash connectivity locally or in CI using:
```bash
# Set your REDIS_URL in .env first
node scripts/verify-upstash.js
```

## 6. Local Production Smoke Test

```bash
npm run build
set NODE_ENV=production
set SESSION_SECRET=replace-with-32-char-secret
set AUDIO_ADMIN_TOKEN=replace-with-long-random-token
set REDIS_URL=redis://localhost:6379
npm start
```

Then open:
- `http://localhost:3000`
