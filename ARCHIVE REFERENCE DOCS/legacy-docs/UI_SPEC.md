# Scholomance V11 — UI Surface Specification

> **Tier 1: Canonical Rules** — binding contracts for component behavior, state, events, and visual structure.
>
> For current file locations, line counts, known bugs, and implementation gaps, see `UI_SPEC_IMPLEMENTATION.md`.

**Arbiter**: Angel (IItsVaelrix) — all exceptions require arbiter sign-off.

---

## 1. World-Law Foundation

> "Every visual element earns its place by being semantically connected to the world's laws."

Scholomance is a ritual-themed text combat MUD where words are weapons. The UI is not a wrapper around the mechanics — it is the world made visible. Every surface MUST connect to a linguistic or phonemic concept.

| Physical Law | UI Expression | Constraint |
|---|---|---|
| Words have mass | Phoneme chips, syllable counters, density indicators | MUST use JetBrains Mono, MUST show numeric values |
| Rhyme keys are resonance frequencies | Rhyme diagram panels, connection visualizations | MUST use school-colored lines, MUST animate on reveal |
| Alliteration is kinetic force | Color-coded onset clusters, alliteration highlights | MUST group by onset consonant |
| Meter consistency is structural integrity | Line gutter with syllable counts, meter badges | MUST appear in gutter, MUST update on edit |
| Rarity and novelty are entropy | Rarity gems on word cards, mythic/legendary styling | MUST use tiered visual treatment per rarity level |

### Design Anti-Patterns (NEVER do these)

- No decorative elements without world-law connection to phonemic or linguistic logic
- No purple-gradient-on-white generic AI aesthetics
- No visible loading spinners — use skeleton states with thematic shimmer
- No alert/confirm boxes — use in-world notification surfaces (scroll unfurl, glyph pulse)
- No modal dialogs for non-destructive actions
- No inline styles for state changes — use CSS classes + event bus

## 2. Design System

### 2.1 Typography (binding)

| Context | Font | Size Variable |
|---|---|---|
| Scroll/combat text | `Georgia, serif` | `var(--text-xl)` = `clamp(1.5rem, 2vw, 2rem)` |
| Navigation/labels | `Space Grotesk` | `var(--text-sm)` = `clamp(0.8125rem, 0.875vw, 0.875rem)` |
| Data/phoneme/code | `JetBrains Mono` | `var(--text-xs)` = `clamp(0.6875rem, 0.75vw, 0.75rem)` |

### 2.2 School Theming (binding)

Schools: **SONIC** (purple), **PSYCHIC** (cyan), **ALCHEMY** (magenta), **WILL** (orange), **VOID** (zinc).

- All school colors MUST come from CSS custom properties generated by `scripts/generate-school-styles.js`
- Components MUST NOT hardcode school hex values
- Active school MUST be expressed via `--active-school-color`, `--active-school-glow`, `--active-school-h/s/l`
- Vowel-to-school mapping: import from `src/data/schools.js` (`VOWEL_FAMILY_TO_SCHOOL`) — NEVER redefine

### 2.3 Effects (binding)

| Effect | Technique | Constraint |
|---|---|---|
| Aurora background | 3-layer radial gradients with parallax | `pointer-events: none`, z-index Layer 0 |
| Vignette | Fixed radial gradient | `pointer-events: none`, z-index 999 |
| Scanlines | Fixed repeating-linear-gradient | `pointer-events: none`, z-index 1000 |
| Glass morphism | `backdrop-filter: blur()` + semi-transparent bg | Disable during drag for perf |

### 2.4 Motion (binding)

| Animation | Duration | Easing | Notes |
|---|---|---|---|
| Page transitions | 200–400ms | easeInOut | Framer Motion |
| Combat reveals | Spring physics | stiffness: 140, damping: 20 | ScoreReveal |
| WordTooltip ink | 380ms | `[0.22, 1, 0.36, 1]` | Custom bezier |
| Reduced motion | 0ms | — | ALL animations MUST respect `prefers-reduced-motion` via `usePrefersReducedMotion` |

## 3. Z-Index Registry (binding)

All z-index values MUST fall within these layers. New values MUST NOT be invented outside this registry without updating this spec.

```
LAYER 0 — Background           z: -1
  Aurora, decorative backgrounds. pointer-events: none.

LAYER 1 — Content               z: 1–50
  Page content, textarea/overlay sync, local stacking contexts.
  Sacred technique: textarea z:1, overlay z:2.

LAYER 2 — Elevated Content      z: 100–200
  Floating panels (via zIndex prop). Default: 100.
  Analysis panels, tool sidebars.

LAYER 3 — Atmosphere            z: 999–1000
  Vignette (999), Scanlines (1000). pointer-events: none.
  These intentionally blanket all content — interactive chrome
  at Layer 4+ punches through them.

LAYER 4 — Interactive Chrome    z: 1100–1500
  Mobile tab bar (1100), mobile nav overlay (1100),
  primary nav (1200), WordTooltip (1300),
  mobile bottom sheet backdrop (1400), mobile bottom sheet (1500).

LAYER 5 — Top-level Overlays    z: 2000
  IDE search overlay, panel drag overrides.

LAYER 6 — A11y / Autocomplete   z: 9999
  Skip link, IntelliSense autocomplete.
  MUST be topmost — no other layer may exceed 9999.
```

**Rules:**
- Values within a layer MAY be any integer in that layer's range
- `!important` on z-index is PROHIBITED — resolve stacking with proper layer assignment
- Inline `zIndex` styles are permitted ONLY for dynamically positioned elements (FloatingPanel prop, drag states)
- New components MUST declare their layer in a CSS comment: `/* z-index: Layer 4 — Interactive Chrome */`

## 4. User Flow Definitions (normative)

Each flow defines the required state transitions. Components MUST support all listed states. Error entry points are marked with `⚠`.

### Flow A: Scroll Authorship

```
IDLE → EDITING → TRUESIGHT_ON → ANALYZING → SAVING → SAVED
  ↑         ↑________|                         |
  |_______________________________________⚠ SAVE_FAILED
```

| State | Trigger | UI Requirement |
|---|---|---|
| IDLE | Page load, no scroll selected | Empty state: "Begin a new scroll" with glyph |
| EDITING | User types in textarea | Gutter updates syllable counts in real-time |
| TRUESIGHT_ON | User toggles Truesight | Textarea: `color: transparent; caret-color: gold`. Overlay: visible, colored word buttons. **Precondition for Flow B.** |
| ANALYZING | User selects analysis mode | AnalysisPanel populates from `usePanelAnalysis` |
| SAVING | User triggers save | Skeleton shimmer on save indicator |
| SAVED | `upsertLocalScroll` succeeds | Brief confirmation glyph, then fade |
| ⚠ SAVE_FAILED | `upsertLocalScroll` throws | "The ink did not hold..." flavor text + manual retry |

### Flow B: Truesight Ritual

**Precondition**: Flow A has reached TRUESIGHT_ON state.

```
WORD_HOVER → TOOLTIP_SHOW → PIN → LOOKUP_REQUEST → CARD_FILLED → NAVIGATE → CLOSE
               ↑_______________________________________________|
               |
          ⚠ LOOKUP_FAILED
```

| State | Trigger | UI Requirement |
|---|---|---|
| WORD_HOVER | Pointer enters word button in overlay | Subtle highlight on word button |
| TOOLTIP_SHOW | Click/tap on word button | WordTooltip appears at click position (desktop: floating card, mobile: bottom sheet) |
| PIN | User drags tooltip | Tooltip becomes pinned, stays on screen |
| LOOKUP_REQUEST | `emit('ui:word_lookup_requested')` | Skeleton shimmer in tooltip card sections |
| CARD_FILLED | `runtime:word_lookup_result` received | Ink animation: blur → brightness → crisp (380ms) |
| NAVIGATE | User clicks suggestion (synonym/rhyme) | New lookup request, breadcrumb trail updated |
| CLOSE | Escape key, click outside, or close button | Tooltip exits with scale+fade animation |
| ⚠ LOOKUP_FAILED | `runtime:word_lookup_result:error` | "The archive could not be reached..." + retry button in tooltip |

### Flow C: Combat Cast

```
ARENA_ENTER → INTRO → PLAYER_TURN → CASTING → SPELL_FLYING → SCORE_REVEAL → CONTINUE
                                                                    ↓
                                                          VICTORY / DEFEAT
                                        |
                                   ⚠ CAST_TIMEOUT
                                   ⚠ PHASER_LOAD_FAIL
```

| State | Trigger | UI Requirement |
|---|---|---|
| ARENA_ENTER | Navigate to /combat | Phaser canvas initializes, loading shimmer with `♩` glyph |
| INTRO | `combat:init` emitted | Opponent name + school revealed, HP/MP bars animate in |
| PLAYER_TURN | `state:update` with `state: PLAYER_TURN` | Spellbook enabled, textarea editable, action bar active |
| CASTING | Player submits spell text | Textarea locks, casting animation plays |
| SPELL_FLYING | `player:cast` emitted to Phaser | Phaser animates projectile, text terminal shows cast text |
| SCORE_REVEAL | `anim:player:done` received | ScoreReveal bottom sheet with spring physics. ScoreTrace breakdown. |
| CONTINUE | Score dismissed | Next turn or opponent response |
| VICTORY | Opponent HP ≤ 0 | Victory overlay with XP earned, triggers Flow D if threshold met |
| DEFEAT | Player HP ≤ 0 | Defeat overlay with "return to sanctum" option |
| ⚠ CAST_TIMEOUT | `castPlayerSpell` exceeds 5s | "The words dissipated..." + return to PLAYER_TURN |
| ⚠ PHASER_LOAD_FAIL | Phaser module fails to load | Text-only fallback mode, combat still functional via terminal |

### Flow D: School Unlock Ceremony

```
XP_THRESHOLD → UNLOCK_TRIGGER → CELEBRATION → SCHOOL_AVAILABLE
                                     |
                                ⚠ UNLOCK_DESYNC
```

| State | Trigger | UI Requirement |
|---|---|---|
| XP_THRESHOLD | XP total crosses unlock boundary | No immediate UI — server validates |
| UNLOCK_TRIGGER | `school:unlocked` event received | Screen dims, atmosphere shifts to new school's palette |
| CELEBRATION | Animation begins | School glyph materializes center-screen, aurora shifts to school color, spring physics reveal |
| SCHOOL_AVAILABLE | Animation completes | New school appears in attunement grid (ProfilePage), Truesight gains new color channel |
| ⚠ UNLOCK_DESYNC | Client thinks unlocked but server disagrees | "The rite requires confirmation..." + re-fetch progression state |

## 5. Event Bus Specification (binding for PLANNED events)

### 5.1 CODEx eventBus (`codex/runtime/eventBus.js`)

Pattern: Request/response via named events. UI emits request with `responseEvent`, engine responds on that channel.

#### EMITTED (live)

| Event | Direction | Payload |
|---|---|---|
| `ui:word_lookup_requested` | UI → Engine | `{ word: string, requestId?: string, responseEvent?: string }` |
| `ui:word_analysis_requested` | UI → Engine | `{ word: string, responseEventName: string }` |
| `ui:combat_action_submitted` | UI → Engine | `{ responseEventName: string, ...CombatScoreRequest }` |

#### CONSUMED (live)

| Event | Direction | Payload |
|---|---|---|
| `runtime:word_lookup_result` | Engine → UI | `{ word: string, requestId?: string, data: LexicalEntry\|null, source: string }` |
| `runtime:word_lookup_result:error` | Engine → UI | `{ word: string, requestId?: string, error: string, code?: string }` |

#### PLANNED — UI requirements (Codex must ratify before implementation)

| Event | Direction | UI Payload Requirement | UI Consumer |
|---|---|---|---|
| `scroll:saved` | UI → Server → UI | Request: `{ scrollId?: string, title: string, content: string, responseEvent: string }` / Response: `{ scrollId: string, savedAt: number, version: number }` | ScrollList save indicator, StatusBar |
| `school:unlocked` | Server → UI | `{ schoolId: School, unlockedBy: 'xp'\|'quest', timestamp: number }` | Flow D ceremony, ProfilePage attunement grid |

### 5.2 combatBridge (`src/pages/Combat/combatBridge.js`)

Pattern: Fire-and-forget pub/sub between React and Phaser. No response events.

#### EMITTED (live)

| Event | Direction | Payload |
|---|---|---|
| `combat:init` | React → Phaser | `{ playerName, opponentName, playerHP, playerMP, opponentHP }` |
| `player:cast` | React → Phaser | `{ damage, school, text }` |
| `opponent:cast` | React → Phaser | `{ spell, damage }` |
| `state:update` | Both directions | `{ state, playerHP, opponentHP, playerMP }` |

#### CONSUMED (live)

| Event | Direction | Payload |
|---|---|---|
| `action:inscribe` | Phaser → React | `{}` |
| `action:flee` | Phaser → React | `{}` |
| `anim:player:done` | Phaser → React | `{}` |
| `anim:opponent:done` | Phaser → React | `{}` |

#### PLANNED — UI requirements (Codex must ratify before implementation)

| Event | Direction | UI Payload Requirement | UI Consumer |
|---|---|---|---|
| `combat:resolved` | Engine → UI | `{ outcome: 'victory'\|'defeat'\|'flee', finalScore: number, xpEarned: number, schoolId: School }` | Victory/defeat overlay, XP animation |
| `xp:awarded` | Engine → UI | `{ amount: number, source: string, newTotal: number, levelUp?: { oldLevel: number, newLevel: number } }` | XP bar animation, level-up ceremony |

## 6. Error State Requirements (binding)

All error states MUST use thematic language. Browser-native error UI (alert boxes, default error pages) is PROHIBITED.

### 6.1 Network Error Pattern

All API-dependent surfaces MUST implement this pattern:

1. **On failure**: Show skeleton shimmer with thematic placeholder (school-colored pulse)
2. **After N consecutive failures**: Surface a manual retry affordance with flavor text
3. **MUST NOT** show browser-native error UI, raw error messages, or stack traces (except dev mode via ErrorBoundary)

### 6.2 IDE Error States

| Error | Trigger | Required Surface |
|---|---|---|
| Dictionary offline | `/api/lexicon/*` fails | WordTooltip: "The archive could not be reached..." + retry glyph |
| Analysis engine error | `usePanelAnalysis` returns error | AnalysisPanel: skeleton shimmer, then "The lens has clouded..." |
| Autosave failure | `upsertLocalScroll` throws | StatusBar: "The ink did not hold..." + manual save button |
| Scroll load failure | `getScrollById` returns null | ScrollList: "This scroll has been lost to the void..." |

### 6.3 Combat Error States

| Error | Trigger | Required Surface |
|---|---|---|
| Phaser load failure | Module import fails | Text-only fallback: combat terminal without canvas, full functionality preserved |
| Spell cast timeout | `castPlayerSpell` exceeds 5s | "The words dissipated..." + return to PLAYER_TURN |
| Opponent response timeout | Opponent move exceeds 10s | "Your foe hesitates..." + force opponent action or skip |

### 6.4 Empty States

Every list/collection surface MUST define an empty state. Empty states are NOT errors — they are onboarding surfaces.

| Surface | Empty State |
|---|---|
| ScrollList (no scrolls) | Sparkle glyph + "Begin a new scroll" |
| AnalysisPanel (no analysis) | Star glyph + "Select a word to analyze" |
| SearchPanel (no results) | "No matches found." / "Type at least 2 characters to search." |
| Combat loading | Musical note glyph `♩` + shimmer, `role="status"` `aria-live="polite"` |
| NexusPanel (no selection) | "Select a word to view its resonance" |
| BattleLog (no history) | "The chronicle awaits your first verse..." |

## 8. Component Contracts (binding)

### 8.1 ScrollEditor — The Sacred Workspace

The textarea overlay sync is the most fragile technique in the codebase. Changes require full visual regression.

**Shared properties (MUST match exactly between textarea and overlay):**
- Font: `Georgia, serif`
- Size: `var(--text-xl)`
- Line-height: `1.9`
- White-space: `pre-wrap`

**Truesight states:**

| State | Textarea | Overlay |
|---|---|---|
| OFF | `color: inherit` | `display: none` |
| ON | `color: transparent; caret-color: gold` | Visible, renders colored word buttons from `analyzedWords` |

**Scroll sync:** `textarea.onScroll → overlay.scrollTop = textarea.scrollTop`

**Semantic props (required interface):**
```
isEditable: boolean
isTruesight: boolean
isPredictive: boolean
analyzedWords: Map
activeConnections: array
colorMap: Map
vowelColors: object
analysisMode: 'none' | 'rhyme' | 'scheme' | 'meter' | 'analyze'
onWordActivate: function
onContentChange: function
onCursorChange: function
```

### 8.2 WordTooltip — The Grimoire Card

A grimoire entry displaying a word's linguistic essence.

**Behavior requirements:**
- Desktop: Floating card, draggable (pointer capture), resizable (8 handles)
- Mobile: Bottom sheet with spring physics
- Ink animation on word change: blur → brightness → crisp (380ms)
- Session history navigation (◀ ▶) with breadcrumb trail
- Z-index: Layer 4 (1300)

**Sections (required order):**
1. Header (word name, drag handle)
2. Art frame (school icon, vowel family glyph)
3. Type line (school + part of speech)
4. Definitions (primary + IPA)
5. Suggestions (allies/foes/resonates)
6. Connections (rhyme link list)
7. Astrology (sign badge, clusters, echo matches)
8. Footer (rhyme key, rarity gem, syllable count)

### 8.3 Navigation — The Archway

**Desktop (≥ 768px):** Inline links — `[SCHOLOMANCE] [Watch] [Read] [Combat] [Nexus] [☀]`
**Mobile (< 768px):** Hamburger menu → full-screen overlay with centered links, glow on selection.

Z-index: Layer 4 (1200). Mobile overlay: Layer 4 (1100).

## 10. Accessibility Requirements (non-negotiable)

### 10.1 ARIA

| Element | Requirement |
|---|---|
| Nav | `aria-label="Primary navigation"` |
| Main content | `id="main-content"` |
| Combat status bars | `role="progressbar"` + `aria-valuenow` |
| WordTooltip | `role="dialog"`, `aria-modal` |
| Action buttons | `aria-label` with context |
| Toast notifications | `role="status"`, `aria-live="polite"` |
| Combat loading | `role="status"`, `aria-live="polite"` |

### 10.2 Focus Management

- Skip link: `href="#main-content"` (z: 9999)
- Focus trap in modals/dialogs
- `tabindex="-1"` on main content for programmatic focus
- Visible focus rings: `var(--focus-ring)`

### 10.3 Reduced Motion

ALL animations MUST be wrapped with `usePrefersReducedMotion()`:
```js
const shouldReduceMotion = usePrefersReducedMotion();
const variants = shouldReduceMotion ? reducedVariants : fullVariants;
```

### 10.4 Keyboard Navigation

- All interactive surfaces MUST be keyboard accessible
- Escape MUST close tooltips, modals, and overlays
- Tab order MUST follow visual order
- Combat action bar MUST support keyboard shortcuts

## 11. State Architecture (binding)

### 11.1 Rules

- ALL UI state MUST live in React hooks or context
- No global mutable variables in UI layer

- Components expose semantic props — no implementation details leak through props

## 12. Security (binding)

- All user input rendering uses React's built-in escaping
- `dangerouslySetInnerHTML` MUST be sanitized per `ARCH_CONTRACT_SECURITY.md` — no exceptions
- No `eval()`, `new Function()`, or inline event handlers from data
- Auth tokens in httpOnly cookies only, never localStorage
- No secrets in client-side code
- Allow-list validation, never deny-list
